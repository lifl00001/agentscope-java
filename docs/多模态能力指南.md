# AgentScope Java 多模态能力指南

## 一、架构总览

AgentScope Java 通过统一的 **ContentBlock** 体系处理文本、图像、音频、视频等多模态内容，采用分层设计：

```
用户消息 (Msg)
  └── ContentBlock 列表
        ├── TextBlock       (文本)
        ├── ImageBlock      (图像)
        ├── AudioBlock      (音频)
        ├── VideoBlock      (视频)
        ├── ThinkingBlock   (推理过程)
        ├── ToolUseBlock    (工具调用)
        └── ToolResultBlock (工具结果，可包含上述任意类型)
```

**核心设计思想**：

- **多模态理解**：通过 Formatter 将 ContentBlock 转换为各 LLM API 所需格式，走 Chat Model 原生多模态端点
- **多模态生成**（图片/视频/音频）：通过 Tool（工具调用）机制实现，Agent 在 ReAct 循环中自动判断并调用

## 二、ContentBlock 体系

### 2.1 基类

```java
// agentscope-core/.../message/ContentBlock.java
@JsonTypeInfo(use = JsonTypeInfo.Id.NAME, property = "type")
public sealed class ContentBlock implements State
        permits TextBlock, ImageBlock, AudioBlock, VideoBlock,
                ThinkingBlock, ToolUseBlock, ToolResultBlock {}
```

### 2.2 媒体来源

所有媒体 Block 共享 `Source` 基类，支持两种来源方式：

| 来源类型 | 类名 | 说明 |
|---------|------|------|
| URL 引用 | `URLSource` | HTTP/HTTPS URL 或本地 `file://` 路径 |
| Base64 编码 | `Base64Source` | 包含 `mediaType`（MIME 类型）和 `data`（Base64 字符串） |

### 2.3 支持的 MIME 类型

| 媒体类型 | 支持的 MIME |
|---------|------------|
| 图像 | `image/png`、`image/jpeg`、`image/gif`、`image/webp` |
| 音频 | `audio/mp3`、`audio/wav`、`audio/mpeg` |
| 视频 | `video/mp4`、`video/mpeg` |

## 三、多模态理解（输入）

### 3.1 创建多模态消息

```java
import io.agentscope.core.message.*;
import java.util.Base64;
import java.nio.file.Files;
import java.nio.file.Paths;

// Base64 方式（推荐，兼容性最好）
String base64Image = Base64.getEncoder().encodeToString(
    Files.readAllBytes(Paths.get("image.png"))
);
ImageBlock imageBlock = ImageBlock.builder()
    .source(Base64Source.builder()
        .data(base64Image)
        .mediaType("image/png")
        .build())
    .build();

// URL 方式
ImageBlock urlImage = ImageBlock.builder()
    .source(URLSource.builder()
        .url("https://example.com/image.jpg")
        .build())
    .build();

// 构建多模态消息
Msg msg = Msg.builder()
    .role(MsgRole.USER)
    .content(List.of(
        TextBlock.builder().text("这张图片是什么？").build(),
        imageBlock
    ))
    .build();
```

### 3.2 各提供商支持矩阵

| 提供商 | 图像输入 | 音频输入 | 视频输入 | Formatter | 自动端点路由 |
|--------|:--------:|:--------:|:--------:|-----------|:------------:|
| **DashScope** | `file://` / data URI | 支持 | 支持（带 fps 帧采样） | `DashScopeChatFormatter` | 是 |
| **OpenAI** | URL / data URI | base64 only | URL | `OpenAIChatFormatter` | - |
| **Anthropic** | URL / base64 | 不支持 | 不支持 | `AnthropicChatFormatter` | - |
| **Gemini** | Inline Data | Inline Data | Inline Data | `GeminiChatFormatter` | - |
| **Ollama** | base64 only | 不支持 | 不支持 | `OllamaChatFormatter` | - |

### 3.3 DashScope 自动端点路由

`DashScopeChatModel` 通过 `EndpointType.AUTO` 自动检测模型名称并路由到对应 API：

| 模型名称模式 | 示例 | API 端点 |
|-------------|------|---------|
| 以 `qvq` 开头 | `qvq-72b`, `qvq-max` | 多模态端点 |
| 包含 `-vl` | `qwen-vl-plus`, `qwen3-vl-max` | 多模态端点 |
| 包含 `-asr` | `qwen3-asr-flash` | 多模odal端点 |
| 以 `qwen3.5` 开头 | `qwen3.5-plus`, `qwen3.5-flash` | 多模态端点 |
| 以 `qwen3.6` 开头 | `qwen3.6-plus`, `qwen3.6-flash` | 多模态端点 |
| 其他 | `qwen-plus`, `qwen-max` | 文本端点 |

也可以通过 `endpointType` 强制指定：

```java
DashScopeChatModel.builder()
    .apiKey(apiKey)
    .modelName("qwen-max")
    .endpointType(EndpointType.MULTIMODAL) // 强制多模态端点
    .formatter(new DashScopeChatFormatter())
    .build();
```

### 3.4 Formatter 转换细节

各 Formatter 的 `MessageConverter` 负责将 ContentBlock 转换为对应 API 的格式：

| Formatter | ImageBlock | AudioBlock | VideoBlock | 关键类 |
|-----------|-----------|-----------|-----------|--------|
| DashScope | `DashScopeContentPart.image`（支持 `file://`） | `DashScopeContentPart.audio` | `DashScopeContentPart.video`（支持帧采样） | `DashScopeMediaConverter` |
| OpenAI | `image_url`（URL 或 data URI） | `input_audio`（仅 base64） | `video_url`（URL） | `OpenAIMessageConverter` |
| Anthropic | `ImageBlockParam`（URL 或 base64） | 不支持 | 不支持 | `AnthropicMediaConverter` |
| Gemini | `Part` + `InlineData`（Blob） | `Part` + `InlineData` | `Part` + `InlineData` | `GeminiMediaConverter` |
| Ollama | 添加到 `OllamaMessage.images` 列表 | 不支持 | 不支持 | `OllamaMediaConverter` |

## 四、多模态生成（输出）

多模态生成通过 **Tool（工具调用）** 实现，不直接通过 Chat Model 输出。Agent 在 ReAct 循环中推理判断需要生成媒体时，自动调用对应的工具方法。

### 4.1 DashScope 多模态工具

**工具类**：`io.agentscope.core.tool.multimodal.DashScopeMultiModalTool`

#### 文生图 `dashscope_text_to_image`

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `prompt` | String | 是 | 生成图片的文本提示 |
| `model` | String | 否 | 模型名称，默认 `wanx-v1` |
| `n` | Integer | 否 | 生成数量，默认 1 |
| `size` | String | 否 | 尺寸，默认 `1024*1024` |

**可用模型**：`wanx-v1`、`qwen-image`、`wan2.2-t2i-flash`

## 五、完整使用示例

### 5.1 配置多模态 Agent

```java
import io.agentscope.core.ReActAgent;
import io.agentscope.core.formatter.dashscope.DashScopeChatFormatter;
import io.agentscope.core.memory.InMemoryMemory;
import io.agentscope.core.message.*;
import io.agentscope.core.model.DashScopeChatModel;
import io.agentscope.core.tool.Toolkit;
import io.agentscope.core.tool.multimodal.DashScopeMultiModalTool;

String apiKey = System.getenv("DASHSCOPE_API_KEY");

// 1. 注册多模态工具
Toolkit toolkit = new Toolkit();
toolkit.registerTool(new DashScopeMultiModalTool(apiKey));

// 2. 创建 Agent（使用支持 function calling 的模型）
ReActAgent agent = ReActAgent.builder()
    .name("MultiModalAssistant")
    .sysPrompt("你是一个具有多模态能力的 AI 助手，可以理解和生成图像、音频、视频。")
    .model(DashScopeChatModel.builder()
        .apiKey(apiKey)
        .modelName("qwen-plus")
        .stream(true)
        .formatter(new DashScopeChatFormatter())
        .build())
    .toolkit(toolkit)
    .memory(new InMemoryMemory())
    .build();

// 3. 多模态理解 — 发送带图片的消息
String base64Image = Base64.getEncoder().encodeToString(
    Files.readAllBytes(Paths.get("photo.jpg"))
);
Msg visionMsg = Msg.builder()
    .role(MsgRole.USER)
    .content(List.of(
        TextBlock.builder().text("描述这张图片的内容").build(),
        ImageBlock.builder()
            .source(Base64Source.builder()
                .data(base64Image)
                .mediaType("image/jpeg")
                .build())
            .build()
    ))
    .build();
Msg response = agent.call(visionMsg).block();

// 4. 多模态生成 — 文生图 / 文生视频
Msg createMsg = Msg.builder()
    .role(MsgRole.USER)
    .textContent("生成一张日落时分的海边风景图")
    .build();
Msg result = agent.call(createMsg).block();
// Agent 会自动调用 dashscope_text_to_image 工具
```

### 5.2 处理多模态工具返回结果

```java
// 通过 Hook 拦截工具调用结果
agent.addHook(new PostActingEvent(event -> {
    if (event.getToolResult() instanceof ToolResultBlock result) {
        for (ContentBlock block : result.getOutput()) {
            switch (block) {
                case ImageBlock img -> System.out.println("生成图片: " + img.getSource());
                case VideoBlock vid -> System.out.println("生成视频: " + vid.getSource());
                case AudioBlock aud -> System.out.println("生成音频: " + aud.getSource());
                case TextBlock txt -> System.out.println("文本: " + txt.getText());
                default -> {}
            }
        }
    }
}));
```

### 5.3 切换生成模型

所有多模态工具的 `model` 参数均为可选参数，有默认值。可通过以下方式切换：

**方式一：对话中指定（推荐）**
```
请用 wan2.5-t2v-preview 模型生成一段猫走路的视频
```

**方式二：修改代码默认值**
```java
// DashScopeMultiModalTool.java 第 849-850 行
String finalModel = Optional.ofNullable(model)
    .filter(s -> !s.trim().isEmpty())
    .orElse("wan2.5-t2v-preview"); // 修改默认模型
```

**方式三：自定义 Tool 类**
```java
@Tool(name = "my_text_to_video", description = "使用特定模型生成视频")
public Mono<ToolResultBlock> myTextToVideo(
        @ToolParam(name = "prompt") String prompt) {
    return dashScopeMultiModalTool.dashscopeTextToVideo(
        prompt, "wan2.5-t2v-preview", null, null,
        null, null, null, null, null, null);
}
```

## 六、工具返回值的处理机制

Tool 返回的 `VideoBlock` / `ImageBlock` / `AudioBlock` 通过 `ToolResultBlock.output` 传递。在后续对话轮次中：

1. **Formatter 转换**：`AbstractBaseFormatter.convertToolResultToString()` 会将媒体 Block 转换为文本引用（如 "The returned image can be found at: URL"），因为大多数 Chat 模型无法直接处理工具返回的媒体内容
2. **Ollama 特殊处理**：`OllamaChatFormatter` 支持将工具结果中的图像"提升"为独立的用户消息，使模型能看到图片
